# LLM Vault

Secure credential management for Claude Code. Store secrets in macOS Keychain, use them in commands without exposing them in conversation history.

## The Problem

Developers using AI coding assistants accidentally paste API keys, tokens, and credentials into chat. These get exposed to conversation history, server logs, and training data. LLM Vault prevents this.

## How It Works

Three layers of protection, installed globally for all Claude Code sessions:

**1. Block credential pastes (UserPromptSubmit hook)**
If you paste an API key into Claude, it gets blocked before Claude ever sees it. Supports 25+ credential patterns (OpenAI, AWS, GitHub, Stripe, Anthropic, etc).

**2. Teach Claude secure usage (Skill + SessionStart hook)**
Claude learns to retrieve credentials from Keychain using inline substitution, so the secret value never appears in conversation:

```bash
# Claude constructs this — the value flows through bash, never through chat
API_KEY=$(~/.llm-vault/hooks/vault get OPENAI_API_KEY) \
  curl -H "Authorization: Bearer $API_KEY" https://api.openai.com/v1/models
```

**3. Detect leaks in tool output (PostToolUse hook)**
If a credential value accidentally appears in command output, Claude is warned not to repeat it.

## Install

```bash
git clone https://github.com/longyi-07/local_llm_vault.git
cd local_llm_vault
./install.sh
```

This copies hooks to `~/.llm-vault/hooks/`, merges them into `~/.claude/settings.json`, and installs the `vault` CLI.

## Usage

### Store a credential

```bash
~/.llm-vault/hooks/vault store OPENAI_API_KEY
# Prompts for value (hidden as you type)
```

Or use the menu bar app for a GUI.

### Use it in Claude Code

Just ask Claude to do something that needs the credential:

> "Make a curl request to the OpenAI API to list models"

Claude will automatically use the secure inline pattern. The credential value never appears in your conversation.

### Manage credentials

```bash
~/.llm-vault/hooks/vault list              # Show stored key names
~/.llm-vault/hooks/vault check KEY_NAME    # Verify a key exists
~/.llm-vault/hooks/vault remove KEY_NAME   # Delete a credential
```

### Pause / Resume

```bash
touch ~/.llm-vault/paused    # Pause all protection
rm ~/.llm-vault/paused       # Resume protection
```

Or use the menu bar app toggle.

### Uninstall

```bash
./uninstall.sh
```

## Menu Bar App

A native macOS menu bar app for managing credentials with a GUI.

- Add, view, update, and delete credentials
- Pause/resume protection with one click
- See credential metadata (created date, last used, use count)
- Starts at login

Build from Xcode:

```bash
open MenuBarApp/LocalLLMVault/LocalLLMVault.xcodeproj
# Press ⌘R to build and run
```

## Architecture

```
~/.llm-vault/                          # Installed location
├── hooks/
│   ├── block_secrets.py               # UserPromptSubmit — blocks pasted credentials
│   ├── check_leaks.py                 # PostToolUse — detects leaked values in output
│   ├── session_start.py               # SessionStart — injects secure usage instructions
│   ├── vault                          # CLI wrapper for Keychain operations
│   └── vault.py                       # Python CLI (alternative)
├── keys.json                          # Credential name registry (names only, never values)
├── metadata/                          # Usage tracking (created, last used, count)
└── paused                             # Exists when protection is paused

~/.claude/settings.json                # Hooks wired here (global, all projects)
~/.claude/skills/llm-vault/SKILL.md    # Behavioral instructions for Claude
```

Credentials are stored in macOS Keychain (hardware-encrypted). The hooks and the menu bar app both access the same Keychain service (`llm-vault`) independently — no IPC, no sockets, no background processes required.

## Security Model

| What | Protected? |
|---|---|
| Credentials in conversation history | Yes — blocked at input, never echoed |
| Credentials in Anthropic server logs | Yes — values pass through bash env, not chat |
| Accidental pastes into chat | Yes — UserPromptSubmit blocks them |
| Credential values in tool output | Yes — PostToolUse detects and warns |
| Credentials at rest | Yes — macOS Keychain (hardware encryption) |

### What this does NOT protect

- Malicious code generated by Claude (always review)
- Credentials already committed to git (use Gitleaks)
- Compromised macOS account (Keychain is accessible)

## Requirements

- macOS (Keychain required)
- Claude Code
- Python 3.7+
